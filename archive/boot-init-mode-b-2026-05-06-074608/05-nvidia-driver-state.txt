# Block all paths that would auto-load NVIDIA modules. The GPU is loaded
# explicitly by aorus-5090-compute-load-nvidia.service, which uses
# `modprobe --ignore-install` to bypass the install lines below.
#
# Why both blacklist and install:
#   blacklist alone is honoured by udev/modalias autoload but ignored by
#   explicit `modprobe nvidia` calls (e.g. by NVIDIA's own scriptlets).
#   `install ... /bin/false` makes any explicit `modprobe nvidia` a no-op
#   unless `--ignore-install` is passed.
#
# nvidia_drm is additionally pinned to no-modeset / no-fbdev so that, even if
# something forces it loaded, it will not register a DRM/KMS device that GNOME
# could try to use as a display output.

blacklist nvidia
blacklist nvidia_modeset
blacklist nvidia_uvm
blacklist nvidia_drm
options nvidia_drm modeset=0 fbdev=0

install nvidia /bin/false
install nvidia_modeset /bin/false
install nvidia_uvm /bin/false
install nvidia_drm /bin/false

# Set /dev/nvidia* permissions at module-load time so that nvidia-modprobe
# (called implicitly by nvidia-smi and various libnvidia-ml callers) does
# NOT reset our 0660 root:ollama back to its 0666 root:root default. The
# loader's explicit chmod is belt-and-suspenders, but these options are
# the load-bearing mechanism: they apply to every device-file creation
# performed by nvidia-modprobe, not just the ones we do explicitly.
#
# Note: these only affect /dev/nvidia0 and /dev/nvidiactl (the nvidia
# main module's devices). The nvidia_uvm submodule has no equivalent
# parameters, so /dev/nvidia-uvm + -uvm-tools + nvidia-caps/* still get
# reset to 0666 by nvidia-modprobe. status.sh sections 8d/8e document
# and check for this; Layer 2 (ICD disable) is the actual safety.
#
# GID 968 == 'ollama' on this host (verified by `getent group ollama`).
# If a fresh Fedora install assigns ollama a different GID, status.sh's
